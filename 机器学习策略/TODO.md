# 中证800机器学习策略后续 TODO

更新时间：2026-05-10

## 当前策略快照

本目录当前归档的是中证800月调仓机器学习策略观察包。主线不是继续大范围重写特征口径，而是在已经审计过的 V4 数据/现场特征对齐基础上，围绕三条模型线做可复现、可比较、可实盘观察的迭代：

- `v46 legacy_unsealed`：当前主 baseline，历史复现最好，但 label boundary 不够干净。
- `v54 recall-rerank top5`：进攻型两阶段候选，收益弹性高，但样本短、集中度高。
- `v410 fixed_iter20 rolling5y 2025q4`：较新的 label-safe 单模型候选，更适合作为未来干净主线。

统一回测约束：中证800股票池、月初调仓、top30 到行业分散 top10 或 recall40 到 rerank top5、9:40 卖出、9:50 买入、不开止损止盈、不开盘中炸板卖出、固定手续费和滑点。

- [ ] 中证800 baseline 池稳定性复核：按年度统计入池数量、剔除 ST/停牌/上市不足 180 天后的有效样本量、行业覆盖、月度换池率。

- [ ] 中证800核心池 + 卫星池：核心仍为中证800，卫星只允许来自中证1000高流动性 topN，设置卫星权重上限 20%/30%/40% 三档。
- [ ] 剔除微盘/低流动性尾部：即使在中证800内，也测试成交额后 20%、波动异常、连续低成交额股票的剔除效果，判断收益是否依赖不可交易尾部。
- [ ] 行业中性股票池版本：每月按中信/申万行业保留行业内流动性和质量排名靠前股票，降低行业样本不均带来的训练偏差。
- [ ] 宽基替代池：对比 `000300.XSHG`、`000905.XSHG`、`000852.XSHG`、`000906.XSHG`、中证800+1000增强池，明确模型 edge 来自大中盘、成长/价值还是小盘流动性溢价。
- [ ] 实盘可交易池：剔除科创/北交/创业板可选开关、剔除无法融入账户权限的股票、剔除长期一字板和高停牌风险股票，形成实盘版本 universe。
- [ ] 股票池漂移监控：每月记录新增/剔除股票，观察新股进入、行业权重变化、指数成分调整是否造成信号跳变。

## P1：特征和数据路线

- [ ] 因子去冗余 v2：当前相关阈值 `0.70` 可以扩展为分行业、分年度相关矩阵，避免只在全样本上去冗余。

- [ ] 市值/流动性显式暴露：加入总市值、自由流通市值、成交额、换手率的中性化版本，区分 alpha 和 size/liquidity beta。

- [ ] 停复牌和涨跌停特征：把过去 20/60 日停牌、涨停、跌停、一字板、开板失败等指标作为模型特征或交易过滤信号。
- [ ] 市场状态特征：扩展 `ctx_market_*`，加入指数趋势、波动分位、市场宽度、成交额分位、大小盘强弱、行业扩散度。
- [ ] 特征重要性稳定性：对每个模型导出 split/gain importance、SHAP 或 permutation importance，并按训练窗口比较漂移。
- [ ] 数据版本化：为每份训练 CSV 输出 manifest，包含日期范围、样本数、列 hash、JoinQuant 数据时间、构建参数。

## P1：Label 和目标函数路线

- [ ] label boundary 全面清理：把 legacy unsealed 只作为观察锚点，新增 label-safe 的 v46 对照，避免未来模型继续混用边界口径。
- [ ] open-to-open / buy-to-next-sell label：构造更贴近 9:50 买入、下月 9:40 卖出的收益标签，减少研究/回测口径错配。
- [ ] 成本后 alpha label：从 label 中扣除估算买卖成本和冲击成本，尤其适合高换手、top5 集中策略。
- [ ] 分位/排序目标：除 regression 外，测试 pairwise rank、Lambdarank、top quantile classifier，观察 top10 命中率是否优于 MSE。
- [ ] 风险调整 label：用未来超额收益除以波动、回撤或 beta 暴露，避免模型只追逐高波动股票。
- [ ] 行业中性 label：用股票收益减行业收益再减 benchmark，检查模型是否主要在押行业轮动。
- [ ] 多周期 label：同时训练 1M、2M、3M alpha，比较信号衰减和调仓频率适配。

## P1：模型策略路线

- [ ] 单模型干净主线：以 `v410 fixed_iter20 rolling5y` 为起点，做 expanding vs rolling5y、fixed_iter20/50、bagging on/off 的严格 OOS 网格。
- [ ] v46 label-safe 复刻：复刻 v46 的特征和组合规则，但使用严格 label-safe，衡量 legacy unsealed 的乐观程度。
- [ ] recall-rerank 稳定性：对 `recall_top_k=30/40/60`、`final_top_n=5/6/8/10`、`prob_topN/prob_x_recall_blend` 做 OOS 稳定性和回撤对照。
- [ ] 模型集成：v46、v410、v54 recall score 做 rank ensemble，测试等权、IC 加权、近期表现加权、市场状态切换。
- [ ] 时间衰减训练权重：对最近 12/24/36 个月样本加权，观察是否改善 2026 OOS。
- [ ] 分市场状态模型：牛市、震荡、熊市或高低波动状态分别训练/选择模型，但必须先做状态定义的先验固定。
- [ ] 分行业/行业组模型：样本足够的行业使用行业内模型，样本不足行业仍用全市场模型，比较行业内 rank 效果。
- [ ] 模型校准：对预测分数做月度横截面 zscore、winsorize、rank normalize，避免极端分数导致组合集中。
- [ ] 过拟合守门：任何新增模型必须通过滚动 OOS、年份拆分、行业拆分、成本敏感性、参数扰动五项检查。
- [ ] 模型降级规则：当新模型连续 N 月弱于 baseline 或触发回撤阈值时，自动回到 v46 或等权 ensemble。

## P1：组合构建路线

- [ ] topN 敏感性：系统测试 top5、top8、top10、top15、top20，比较收益、回撤、换手、行业集中度和持仓重叠。
- [ ] 行业 cap 网格：`industry_cap_ratio=0.15/0.20/0.25/0.30`，判断 20% 是否过紧或过松。
- [ ] 分数加权替代等权：测试等权、rank 权重、softmax 权重、波动率倒数权重、行业内等权。
- [ ] 组合优化器：在不引入复杂黑箱的前提下，做行业、个股、换手、流动性约束下的 score maximization。
- [ ] 持仓缓冲带：已有持仓只要仍在 top20/top30 就不卖，降低换手和成本，和纯月度全换仓对比。
- [ ] 交易禁入名单：连续涨停、低成交额、财报异常、监管风险、近期大幅减持等作为组合层过滤，而不是模型特征硬学。
- [ ] 个股权重上限：top5 策略尤其需要 15%/20%/25% 上限压力测试，避免单票冲击过大。
- [ ] 行业暴露预算：设置相对中证800行业偏离上限，输出牺牲收益和降低回撤的 tradeoff。
- [ ] 与 ETF/指数风控叠加：当模型股票信号弱或市场状态恶化，部分仓位切到宽基/货币/空仓，测试是否改善尾部风险。

## P1：交易执行和成本路线

- [ ] 成交时点敏感性：比较 9:35、9:50、10:30、14:30 买入，以及开盘卖/收盘卖对收益的影响。
- [ ] 滑点压力测试：当前 `PriceRelatedSlippage(0.00246)` 外，再测 10bp、25bp、50bp、100bp 和按成交额分层冲击成本。
- [ ] 涨跌停处理归因：每月记录目标买不进、持仓卖不出、昨日涨停延迟卖出的收益影响。

- [ ] 容量估算：按目标股过去 20 日成交额的 1%/3%/5% 参与率估算策略容量和冲击成本。
- [ ] 换手归因：区分模型换手、股票池换手、行业 cap 换手、涨跌停导致的被动换手。
- [ ] 实盘订单模板：把 JoinQuant 回测脚本和 QMT/实盘执行需要的目标持仓 CSV、订单拆分、失败重试格式打通。

## P1：风险和绩效归因路线

- [ ] 年度/季度拆分：每条模型输出年度收益、最大回撤、月胜率、最差月份、最长回撤修复期。
- [ ] 行业归因：每月持仓行业权重、行业收益贡献、相对 benchmark 行业偏离。
- [ ] 风格归因：市值、估值、质量、动量、波动、流动性暴露，确认收益是否主要来自某个拥挤风格。
- [ ] 个股集中归因：统计前 10 大贡献股和拖累股，避免单票事件解释大部分超额。
- [ ] 市场状态归因：按指数趋势、波动分位、市场宽度分组评估策略表现。
- [ ] 回撤复盘模板：对每次超过 5%/8%/10% 的回撤，归因到市场、行业、模型、执行、个股事件。
- [ ] benchmark 替换：除中证800外，对沪深300、中证500、中证1000做相对收益归因，识别隐含风格 beta。

## 建议执行顺序

1. 先做 P0 的模型注册表、结果归档、严格 OOS 对照和复现文档。
2. 第二步推进股票池实验，优先比较纯中证800、中证800+1000高流动性、核心卫星三条。
3. 第三步推进 label-safe v46、v410 干净主线和 v54 topN/recallK 稳定性。
4. 第四步在确认 alpha 稳定后再做组合优化、交易成本、实盘执行工程化。
5. 任何新增复杂模型都必须先通过可复现、严格 OOS、成本压力和风险归因四道检查。
