# 机器学习选股 Alpha 攻关 TODO

更新时间：2026-05-10

## 目标

这份 TODO 聚焦一件事：在现有月度机器学习选股框架上，找到更强、更稳、更可交易的 alpha。当前中证800只是已归档 baseline，不应成为边界；后续可以扩到中证1000、中小盘高流动性池、全 A 可交易池，甚至按市值带/行业带做专门模型。

现有锚点：

- `v46 legacy_unsealed`：主 baseline，历史表现最好，但 label boundary 不够干净。
- `v54 recall-rerank top5`：已验证过的“召回 + 精排”思路，值得继续泛化。
- `v410 fixed_iter20 rolling5y`：较干净的 label-safe 单模型观察线。

核心评价：严格 OOS 的超额收益、回撤、月胜率、最差月份、换手后收益、容量、相对 baseline 的稳定提升。监控和工程化只保留最小必要项。

## P0：先做最可能提高 alpha 的实验

- [ ] **小中盘高流动性主线**：构建 `CSI500 + CSI1000 + 非指数高流动性中小盘` 股票池，剔除 ST、上市不足 180 天、低成交额、长期停牌、一字板高发股票。假设：现有因子在中证800内被大盘股稀释，中小盘里质量/动量/流动性错价更强。
- [ ] **中证800 vs 中证1000 vs 中小盘全 A 分层训练**：不要只换回测股票池，要分别训练模型；大盘、中盘、小盘的有效因子和噪声结构不同。输出每个市值带的 IC、top10 收益、回撤和换手。
- [ ] **Recall-Rerank 2.0**：保留你的两阶段思路，但把第一阶段改成“高召回 alpha 候选”，第二阶段改成“可交易胜率/尾部风险过滤”。第一阶段目标是不要漏掉强票，第二阶段目标是少选炸雷、买不进、冲高回落票。
- [ ] **Learning-to-Rank 版本**：用 LightGBM/XGBoost ranking objective 替代回归 MSE，把每个月作为一个 query，直接优化月度横截面排序；对比 regression、binary top20 classifier、LambdaMART 三种目标。
- [ ] **Meta-Labeling 版本**：第一模型只决定候选方向和排序，第二模型判断“这只候选是否值得下注/加权”。label 可定义为入选后是否跑赢候选池中位数、是否进入当月 top20%、或是否超过成本后 alpha 门槛。
- [ ] **TopK Hit Rate 模型**：把目标从预测收益数值改成预测“未来一个月是否进入池内 top10/top20/top30”。月度选股只关心排序头部，MSE 最小不一定等于头部命中率最高。
- [ ] **组合级 rerank**：第二阶段不只给单票打分，而是逐只贪心加入组合，目标最大化 `预测alpha - 行业集中惩罚 - 风格拥挤惩罚 - 换手成本 - 流动性惩罚`。
- [ ] **多模型 rank ensemble**：把 v46、v410、v54 recall、ranking 模型、meta-label 模型做 rank ensemble。先测简单等权，再测按最近滚动 OOS IC/命中率加权。

## P0：股票池和适用域

- [ ] **中小盘高流动性池**：每月从全 A 中选自由流通市值分位 20%-80%、过去 60 日成交额分位前 60%、停牌天数少、上市满 180 天的股票。目标是吃中小盘 alpha，但避开微盘流动性陷阱。
- [ ] **中证1000/中证2000风格池**：若 JoinQuant 数据支持，优先测试中证1000；再测试更小市值但加入更严格成交额和涨跌停过滤。比较收益是否来自小盘 beta 还是模型真 alpha。
- [ ] **核心-卫星池**：核心池用中证800/中证1000稳定成分，卫星池用全 A 高流动性中小盘。训练和选股时允许卫星更进攻，但组合层设置卫星权重上限。
- [ ] **行业专门池**：对医药、电子、机械、汽车、基础化工、有色、传媒等 alpha 差异大的行业单独训练行业内 ranking 模型，再汇总到组合层。
- [ ] **排除低 alpha 区域**：如果大金融、公用事业、超大市值央企长期贡献低或更像防御 beta，测试从进攻模型里剔除，只在风控模型中保留。
- [ ] **壳价值/困境反转池**：单独做小市值低估值、高现金流改善、低机构覆盖的反转池；这条线和质量成长模型分开训练，不混成一个平均模型。

## P1：模型结构新方案

- [ ] **Cascade 多阶段漏斗**：`全池 -> recall top100 -> quality/risk filter top60 -> rerank top20 -> portfolio optimizer top8/10`。每一层优化不同任务，避免单模型同时学收益、风险和交易可行性。
- [ ] **Diversity Rerank**：借鉴搜索排序里的多样性 rerank，候选股分数相近时优先选择行业、风格、相关性更分散的股票，减少 top5/top10 策略单一风格暴露。
- [ ] **Residual Alpha Model**：先用传统风格因子解释收益，如市值、估值、动量、行业、波动、流动性，再训练模型预测残差 alpha。目标是减少模型偷吃小盘/行业 beta。
- [ ] **Mixture of Experts**：按市值带、行业组、市场状态训练多个专家模型，再用 gating model 决定每月每只股票更信哪个专家。
- [ ] **Market-Regime Switcher**：用市场宽度、指数趋势、成交额、波动率、大小盘强弱定义状态；不同状态切换 value/quality/momentum/liquidity 权重或切换模型。
- [ ] **Temporal Sequence Model**：把单月截面特征扩展为过去 6-12 个月特征序列，测试 TCN/GRU/Transformer encoder 是否能捕捉因子改善、动量衰减和财务趋势。先做轻量版本，避免样本太小。
- [ ] **Autoencoder 风格压缩**：用 autoencoder/PCA 从大量因子中学习低维隐含风格，再把 reconstruction error 或 latent factors 加入模型，寻找传统风格之外的错价。
- [ ] **Pairwise Preference Model**：构造同月股票对，学习 A 是否优于 B，最后用 pairwise 胜率排序。适合处理收益噪声大、绝对 alpha 难预测的问题。
- [ ] **Quantile/Distribution Model**：不只预测均值，也预测上行分位和下行分位；选“上行高、下行受控”的股票，而不是均值最高但尾部很差的股票。
- [ ] **Stacking with Linear Risk Head**：底层用树模型/神经网络抓非线性，上层用简单线性模型控制行业、市值、波动和流动性暴露，兼顾 alpha 与可解释性。

## P1：Label 设计

- [ ] **交易口径 label**：把 close-to-close 改为接近实盘的 `调仓日 9:50 买入 -> 下月调仓日 9:40 卖出`，至少用 open/high/low/close 近似，减少研究和回测错配。
- [ ] **成本后 top label**：定义 `alpha_after_cost > 0`、`进入当月前 20%`、`跑赢行业中位数` 三套 label，同时比较哪套最能提升组合收益。
- [ ] **行业内 rank label**：每只股票先和同行业比较，再汇总跨行业选择，避免模型靠行业行情而非个股 alpha。
- [ ] **小盘专用 label**：对中小盘加入涨跌停、停牌、成交额冲击惩罚，防止模型选到纸面收益高但买卖困难的票。
- [ ] **多周期融合 label**：同时预测 1M、2M、3M alpha；月调仓仍按 1M 执行，但优先选 1M 强且 3M 不差的股票，降低短期噪声。
- [ ] **Drawdown-Aware label**：未来一个月收益高但期间最大回撤很深的股票降权，尤其适合集中 top5/top10。
- [ ] **Event-aware label**：财报季、业绩预告、分红除权、解禁、减持窗口单独打标，判断模型是否在事件月份失效或有额外 alpha。

## P1：特征增量

- [ ] **中小盘特色因子**：成交额改善、换手率变化、量价背离、涨停后回踩、低位放量、波动收缩、机构覆盖低、财务质量改善。
- [ ] **财务趋势因子**：收入/利润/现金流同比和环比改善、毛利率改善、经营现金流修复、应收/存货压力下降。重点看“改善”而不只是静态估值。
- [ ] **分析师/预期差因子**：如果数据可得，加入盈利预测上调、评级变化、研报覆盖变化；小中盘里预期差可能比静态质量更有效。
- [ ] **资金行为因子**：北向、融资融券、龙虎榜、主力净流入、机构持仓变化等，数据可得时作为单独特征组做消融。
- [ ] **微结构特征**：开盘跳空、早盘强弱、尾盘拉升、日内振幅、过去 N 日涨跌停打开情况，用于 rerank/交易过滤而不是全塞进 baseline。
- [ ] **拥挤度因子**：同质动量拥挤、行业拥挤、短期涨幅拥挤、成交额异常拥挤；用于避免模型在热门风格末端追高。
- [ ] **因子交互特征**：质量 × 低估值、动量 × 低波动、盈利改善 × 成交额改善、小市值 × 高流动性。Gu-Kelly-Xiu 类研究提示非线性交互通常是树模型优势来源。
- [ ] **另类数据占位**：公告文本情绪、互动易/新闻热度、研报摘要、产业链主题热度可作为后续支线，但必须单独验证增量，避免引入噪声。

## P1：组合和持仓优化

- [ ] **Alpha-First Optimizer**：目标函数优先最大化预测 alpha，约束只保留必要的行业上限、个股上限、流动性下限和换手上限，不要过早做过度中性化。
- [ ] **TopN 动态选择**：市场宽度高、模型分数分散度高时 top5/top8；市场差或分数差距小时 top10/top15，降低集中踩雷。
- [ ] **Score Gap 规则**：如果第 5 名和第 15 名分数差距很小，说明头部不确定性高，组合应更分散；如果差距很大，可以更集中。
- [ ] **持仓缓冲带**：已有持仓仍在 top20/top30 且风险不过线就保留，减少高换手吞噬 alpha。
- [ ] **行业内先选、全局再配**：每个行业先选本行业强票，再按全局 alpha 和行业机会度分配名额，避免一个强行业挤掉所有名额。
- [ ] **小盘容量约束**：单票目标成交额不超过过去 20 日均成交额的 1%-3%，超出则自动降权或替换。
- [ ] **组合相关性惩罚**：候选股过去 60 日收益相关性太高时，只保留分数最高的一部分，减少同涨同跌。

## P2：验证框架，只保留必要项

- [ ] **滚动 OOS 对照**：每个新方案必须和 v46/v410/v54 在相同区间、相同成本、相同股票池上比较。
- [ ] **消融实验**：股票池、label、特征、模型、组合规则一次只改一个主变量，避免不知道 alpha 来自哪里。
- [ ] **年份/市场状态拆分**：至少按牛/熊/震荡、高低波动、大小盘强弱拆分，确认不是单一年份偶然有效。
- [ ] **成本压力测试**：10bp、25bp、50bp、100bp 滑点；中小盘策略必须额外做成交额冲击。
- [ ] **容量估算**：每个策略给出 100万、500万、1000万、3000万资金规模下的可成交性和收益折损。
- [ ] **最小监控**：只保留月度 OOS 收益、回撤、换手、行业/市值暴露、买不进卖不出次数；不扩展成大工程。

## 优先执行顺序

1. 做中小盘高流动性股票池，并复刻现有 v46/v410/v54，确认 alpha 是否更强。
2. 做 Learning-to-Rank 和 Meta-Labeling 两条新模型线，直接对比现有 recall-rerank。
3. 把 recall-rerank 升级成多阶段 cascade：收益召回、风险过滤、组合 rerank。
4. 增加交易口径 label 和行业内 rank label，解决研究收益与实盘回测错配。
5. 如果前四步有稳定增量，再测试 Mixture of Experts、sequence model、autoencoder 等更复杂结构。

## 可参考的研究脉络

- Gu, Kelly, Xiu, [Empirical Asset Pricing via Machine Learning](https://www.nber.org/papers/w25398)：树模型和神经网络在横截面收益预测上常优于线性模型，关键在非线性和交互项。
- Burges, [Learning to Rank on a Cluster using Boosted Decision Trees](https://www.microsoft.com/en-us/research/publication/learning-to-rank-on-a-cluster-using-boosted-decision-trees/)：把每个月当 query，直接优化排序，适合“只关心 topK 股票”的选股问题。
- López de Prado, [Machine Learning for Asset Managers](https://www.cambridge.org/core/books/machine-learning-for-asset-managers/6D9211305EA2E425D33A9F38D0AE3545)：第一模型产生候选，第二模型决定是否下注或如何加权，适合作为 recall-rerank 的泛化版本。
- Gu, Kelly, Xiu, [Autoencoder Asset Pricing Models](https://papers.ssrn.com/sol3/papers.cfm?abstract_id=3335536)：用隐含因子和非线性 exposure 压缩高维特征，可作为风格残差和 regime 表征支线。
- Chen, Pelger, Zhu, [Deep Learning in Asset Pricing](https://arxiv.org/abs/1904.00745)：深度模型可以学习时变状态和非线性条件资产定价，但应放在样本量和 OOS 纪律足够后再做。
