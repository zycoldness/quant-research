# Model Files

Place the three model pkl files in this folder or upload them next to the corresponding JoinQuant backtest script.

Expected files:

- `model_candidate_v46_lgb_direct_hybrid_l2_ff10_2019_2025q1_legacy_unsealed.pkl`
- `model_candidate_v54_portfolio_oof_rerank_v410_q1.pkl`
- `model_candidate_v410_fixed_iter20_rolling5y_l2_ff10_2025q4.pkl`

The pkl files were not present in the local workspace when this package was assembled, so they are referenced by exact filename instead of copied.

Refreshed training exports generated by:

```text
notebooks/中证800月调仓候选模型导出_2019_20251231新版训练.ipynb
```

will use a date-derived suffix by default:

```text
train20190101_20251231_labelend20260531
```

This is intentional so refreshed models do not overwrite the three core observation pkl files above.
